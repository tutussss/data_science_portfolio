How to do it...

To begin the recipe, we need to log into AWS and take the following steps:

  1. Create a Virtual Private Cloud (VPC).
  2. Navigate to the Amazon VPC console.
  1. Click on Launch VPC Wizard, as follows:

  1. Select VPC with a Single Public Subnet, as follows:

Note: When we are creating VPC we should make sure, that we have Public Subnet with Internet Gateway and Router table. Otherwsise, you will be able access Tableau Server GUI only from VPC. You can read more about this at AWS documentation: https://docs.aws.amazon.com/vpc/latest/userguide/what-is-amazon-vpc.html. In this chapter, we didn't focus on creating Internet access for the server and used RDP and Private DNS to access Tableau Server.
  1. Give the VPC a name and click Create VPC, as follows:

  1. We will see our VPC created, as follows:

  1. The next step will be to configure the networking and security. To configure networking and security, we will use the following steps:
    • Go to the EC2 console.
    • Ensure your region is in the same location as the VPC we just created.
    • Click Security Groups in the Navigation pane.
    • Click on Create Security Group.
    • Fill in the security group name, description, and select the VPC we created in the previous steps, as follows:

  1. Create the inbound rules, by clicking on Add Rule, as follows:

  1. Create the inbound traffic rules for HTTP, HTTPS, and RDP. Then, click on Create, as follows:

  1. Use Custom to specify an IP range or another security group. For production environments, it is important to limit who has access to your environment. In addition, we should add SSH port 22 for the Linux environment. Finally, we need to add port 8850 for the Tableau Service Manager (TSM) web interface.
  2. Launch an Amazon EC2 instance.
  3. Navigate to the EC2 console.
  4. Verify you are in the same region as the VPC created previously.
  5. Under Create Instance, Click on Launch Instance, as follows:

  1. Select an Amazon Machine image for Windows Server 2012 R2, 64-bit or Windows Server 2016, 64-bit, as follows:

  1. Within the Choose an Instance Type menu, scroll down to select an instance that meets the recommended requirement for Tableau Server: 16+ vCPU and 64+ GB RAM (4 GB RAM per vCPU), as follows:

Installing Tableau on Amazon EC2 T2 instances are not supported. Typical environment types and sizes for development, test, and production are as follows: c5.4xlarge, m5.4xlarge, r5.4xlarge
  1. Configure the instance type by choosing the VPC created in the previous steps, as follows:

  1. Add another 100 GB of storage as a separate drive. You may have to partition and mount this drive, as follows:

  1. In the Configure Security Group menu, choose the security group we created in the previous recipe steps, as follows:

  1. Review the instance you created and then launch it.
  1. Create a key pair file (or use an existing one). We need to create this .pem file in order to log in to the server remotely, as follows:

  1. It can take a few minutes to launch the instance. While it is initializing, copy the instance ID for the next step, as follows:

  1. Create an elastic IP address for the VPC:
  • Navigate to the Amazon VPC console
  • Use the same region as the VPC created in earlier steps
  • In the Navigation pane, choose elastic IP, as follows:

  1. Click on the Allocate new address button and then click on Allocate in the next screen, as follows:

  1. Once created, click on the Actions menu, and select Associate address, as follows:

  1. In the Associate address window, choose the Instance resource type and the server/instance you created in the earlier steps, as follows:

  1. Log into Amazon EC2 using remote desktop:
    • Navigate to the EC2 console.
    • Select the region where we launched the instance.
    • In the EC2 dashboard, click on Instances.
    • Select the instance, click on Actions, and choose Connect, as follows:

  1. In the Connect To Your Instance dialog box, click on Download Remote Desktop File.
  2. Click on Get Password and select the .pem file created earlier.
  3. Click on Decrypt Password, when it's displayed copy it and keep it, as follows:

  1. Log in using the .rdp file saved earlier and ignore any messages or warnings about unknown publishers or unknown connections.
  2. Install Tableau Server as follows:
    • Download the installer for Tableau Server 2019.x and save it locally
    • For a single node, guided install, double-click on the installer and follow the instructions to create a new Tableau Server installation. Rather than installing onto the C drive, it is best practice to install the application on a separate drive. We added this in the previous steps for setting up an EC2 instance
  1. Once Tableau Server has been installed, you must use an account with local administrator rights to run Tableau Services Manager (TSM) Web UI, and CLI tools. Follow the guided instructions to activate and register Tableau Server. Go through the following setups in the following screenshot:
